# Руководство по разработке WebHub-Desktop

Спасибо, что решили заглянуть "под капот" нашего проекта! Это руководство поможет вам разобраться в архитектуре приложения и процессе внесения изменений.

## 🚀 Быстрый старт

Проект построен на базе **Electron**, поэтому для работы вам понадобятся [Node.js](https://nodejs.org/) и npm.

1. **Клонируйте репозиторий:**
   ```bash
   git clone [https://github.com/ваш-аккаунт/WebHub_Desktop.git](https://github.com/ваш-аккаунт/WebHub_Desktop.git)
   cd WebHub_Desktop
   
2. **Установите зависимости:**
```Bash
npm install

3. **Запустите приложение в режиме разработки:**
```Bash
npm start

Архитектура проекта
Приложение разделено на несколько уровней ответственности:

Main Process (src/main.js): Управляет жизненным циклом приложения, созданием окон, глобальными горячими клавишами (например, Alt+Ё для Discord) и системными вызовами.

Preload (src/preload.js): Безопасный мост (ContextBridge) между системными функциями Electron и веб-интерфейсом. Если вы добавляете новую функцию связи (IPC), её канал должен быть внесен в validChannels.

Renderer Process (UI): Построен на базе кастомных веб-компонентов:

tabs.component.js — управление вкладками и логика инъекций в webview.

config.component.js — управление настройками и профилями.

adblocker.js — логика фильтрации контента на базе Ghostery.

🛠 Правила внесения изменений
1. Безопасность (IPC)
При добавлении новых событий между Main и Renderer процессами:

Всегда используйте contextBridge в preload.js.

Не передавайте объект event целиком, только необходимые данные.

Обязательно добавляйте название канала в список разрешенных в preload.js.

2. Взаимодействие с WebView
Для изменения внешнего вида или поведения сайтов (YouTube, VK, Discord) используйте файл src/webview.injections.js.

Стили должны быть атомарными и использовать !important, чтобы перекрывать стили сайтов.

JS-инъекции должны быть обернуты в анонимные функции (function() { ... })().

3. Горячие клавиши
Глобальные хоткеи регистрируются в main.js. Помните, что они перехватывают нажатия во всей системе, поэтому используйте редкие сочетания (например, Alt+Backquote).

🐞 Сообщения об ошибках
Если вы нашли баг:

Проверьте консоль разработчика (Ctrl+Shift+I в приложении).

Опишите шаги для воспроизведения.

Если ошибка касается шрифтов ("Slow network detected"), убедитесь, что в @font-face указан font-display: swap.
